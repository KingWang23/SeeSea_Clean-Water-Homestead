# SeeSea 完整技术报告

## 1. 项目概述

SeeSea 是一套面向海洋垃圾治理行动的 AI 协同系统。项目围绕“现场采集困难、图像质量差、人工表单易错漏、垃圾来源难标准化判断、数据无法持续沉淀”等问题，构建了一条从潜水员端采集到管理员端分析与导出的完整闭环。

项目目标不是只做一个识别 Demo，而是做一个可以真实运行、可持续扩展、适合公益机构试点的治理数据系统。

## 2. 设计目标与评审对应关系

### 2.1 技术实现难度

系统包含多阶段技术链路：

- 水下图像增强

- OCR 与包装线索提取
- 人工表单核对
- 志愿者反馈语义分析
- 扩散路径模拟

- 管理员 AI 指挥简报

这使系统既包含视觉处理，也包含结构化数据处理和文本分析。

### 2.2 创新性

系统创新点主要体现在：

- “图像即数据”

### 2.3 功能完整度

SeeSea 已形成完整业务闭环：

- 移动端采集

- 后台审核与流转

### 2.4 可实施性

## 4. 核心模块与实现方式

### 4.1 登录与权限控制

#### 目标

管理员和潜水员必须具备清晰的角色隔离，同时支持比赛演示时在同一浏览器中并行登录两个角色。

#### 实现

- 登录接口签发 token
- 前端按角色保存 token
- API 请求使用 Bearer Token
- 页面与 API 层按角色校验权限

#### 价值

- 避免旧式共享会话相互覆盖
- 适合双端协同演示

### 4.2 潜水员端采集

#### 目标

让潜水员在手机浏览器中即可完成现场信息采集。

#### 实现

- 拍照上传按钮动态触发相机采集
- 选择图片按钮支持相册与本地文件
- 上传前显示缩略图预览
- 表单支持人工条目录入和反馈补充

#### 技术

- 原生 file input
- 动态 capture 策略
- 浏览器端图片预览

### 4.3 定位与地图选点

#### 问题

在 HTTP 环境下，很多手机浏览器不允许直接调用系统定位。

#### 实现

- 优先使用 `navigator.geolocation`
- 若失败，则切换到 Leaflet 地图选点
- 点击地图后自动写回经纬度

#### 价值

- 保证移动端可用性
- 保证比赛现场演示稳定

### 4.4 图像增强

#### 目标

提升水下图像清晰度和颜色可读性，为识别和展示提供更稳定输入。

#### 实现思路

- 亮度增强
- 对比度增强
- 偏色纠正
- 局部细节强化

#### 当前工程实现

- OpenCV 图像处理流程

### 4.5 垃圾识别

#### 目标

输出垃圾类别、材质、来源活动、数量和证据来源。

#### 架构

SeeSea 当前采用三层识别链路：

1. 传统视觉规则
2. 本地 ONNX 检测器
3. 远程视觉模型辅助

#### 融合逻辑

- 规则提供稳定底线
- ONNX 提供候选检测
- 视觉模型补充复杂场景判断与来源推断

#### 输出字段

- `item_name`
- `category`
- `material`
- `source_activity`
- `size_level`
- `quantity`
- `confidence`
- `evidence`

### 4.6 OCR 与包装线索

#### 目标

增强对垃圾来源的判断能力。

#### 实现

- 提取图像中的包装文字
- 将文本和包装匹配结果作为证据写入结果

#### 价值

- 支撑“垃圾身份证”
- 提高来源判断可解释性

### 4.7 人工表单核对

#### 目标

减少人工录入误差，提高统计可信度。

#### 实现

- 对比 AI 识别条目与人工条目
- 统计总数差异
- 标出漏填项和数量偏差项

#### 价值

- 从经验录入转向证据核验
- 提升项目台账质量

### 4.8 志愿者反馈语义分析

#### 目标

将自然语言反馈转成结构化行动信息。

#### 实现

- 从文本中提取密集度、复杂度和安全风险
- 生成风险等级

#### 价值

- 把经验描述转成可管理信息
- 丰富案例上下文

### 4.9 扩散路径模拟

#### 目标

从“识别垃圾”扩展到“推测来源与扩散倾向”。

#### 实现

- 基于潜点位置和来源类型进行规则驱动模拟
- 输出轨迹点序列

#### 价值

- 为治理决策提供更高层信息
- 支持后续接入真实海流模型

### 4.10 单案例报告与管理员简报

#### 目标

把结构化结果转成适合管理和传播的自然语言内容。

#### 实现

- 单案例分析报告
- 潜水员汇总报告
- 管理员 AI 指挥简报
- 支持远程 LLM 和本地规则双模式

#### 容错

- 超时、重试、错误透传
- 连通性测试
- 远程失败时自动降级到本地规则

### 4.11 管理员看板

#### 功能

- 指标卡片
- 地图总览
- 趋势图
- 分布图
- 同源预警
- AI 指挥简报
- 案例审核流
- 数据导出

#### 价值

SeeSea 因此不是单一识别工具，而是一个治理支持系统。

## 6. 创新性分析

SeeSea 的创新不在于单一模型，而在于把多种技术组织成公益机构真正能用的工作流。

### 创新 1：图像即数据

潜水员拍到的图片直接进入数据链路，而不是停留在展示层。

### 创新 2：垃圾身份证

每件垃圾都可以被标注为类别、材质、来源、数量和证据来源。

### 创新 3：AI 与人工双向校验

AI 不替代人工，而是帮助人工纠错和补全。

### 创新 4：从案例级走向行动级

系统不仅产出单案例结果，还能产出管理员简报、风险提醒和导出数据。
