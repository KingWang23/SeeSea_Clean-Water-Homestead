# SeeSea 详细使用说明

## 1. 环境准备

```bash
cd "CCF 海洋马拉松/Project"
conda activate clean_underwater_demo
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

电脑本机访问：

- `http://127.0.0.1:8000`

手机访问：

- `http://电脑局域网IP:8000`

不要使用 `https://`，否则移动端可能报安全连接错误。

## 2. 默认账号

- 管理员：`admin / admin123`
- 潜水员 A：`diver_a / diver123`
- 潜水员 B：`diver_b / diver123`

## 3. 潜水员端使用流程

### 3.1 登录

1. 进入登录页
2. 输入潜水员账号密码
3. 进入潜水员工作台

### 3.2 新建案例

按顺序填写：

1. 实例标题
2. 潜点名称
3. 拍摄日期
4. 位置描述
5. 经度纬度
6. 天气与能见度
7. 人工表单备注
8. 志愿者语义反馈

### 3.3 上传图片

系统提供两个按钮：

1. `拍照上传`
2. `选择图片`

推荐：

- 手机端优先点击 `拍照上传`
- 电脑端或已拍好的图片使用 `选择图片`

选择图片后，页面会显示预览。

### 3.4 定位方式

优先级如下：

1. 点击 `读取手机定位`
2. 若失败，点击 `地图选点`
3. 在地图上点击目标位置
4. 系统会自动写入经纬度

说明：

- 在 `HTTP` 页面下，很多手机浏览器会拒绝系统定位
- 这是浏览器安全策略，不是系统故障
- SeeSea 已提供地图选点作为稳定替代方案

### 3.5 人工表单录入

1. 点击 `新增表单条目`
2. 填写垃圾名称
3. 填写数量
4. 选择状态

### 3.6 提交

点击 `提交本次实例` 后，系统会自动执行：

1. 图片保存
2. 图像增强
3. AI 垃圾识别
4. 人工表单核对
5. 志愿者反馈语义分析
6. 扩散路径模拟
7. 案例报告生成

### 3.7 查看结果

潜水员端可查看：

- 原图
- 增强图
- AI 识别图
- 识别与核对结果
- 案例分析报告
- 管理员反馈
- AI 补充洞察

### 3.8 删除案例

1. 打开某个案例
2. 点击 `删除本案例`
3. 确认删除

## 4. 管理员端使用流程

### 4.1 登录

使用管理员账号登录后进入后台。

### 4.2 查看总览

管理员首页可查看：

- 指标卡片
- 类别/材质/来源分布
- 月度趋势
- 潜点地图
- AI 指挥简报
- 风险预警

### 4.3 筛选案例

支持按以下维度筛选：

- 关键词
- 风险等级
- 审核状态
- 清理状态
- 潜水员

### 4.4 查看案例详情

管理员可查看：

- 原图 / 增强图 / AI 识别图
- AI 与人工核对
- 污染源分析
- 扩散路径
- 相似案例
- 错误信息

### 4.5 审核工作流

管理员可修改：

- 审核状态
- 清理状态
- 优先级
- 管理员备注

### 4.6 重新生成分析

如案例内容更新或需要重新跑模型，可点击：

- `重新生成分析`

### 4.7 删除案例

如为误传、重复案例或测试样本，可点击：

- `删除案例`

### 4.8 导出

支持：

- `导出 Markdown`
- `导出 JSON`
- `导出 CSV`

### 4.9 潜水员汇总

管理员点击某位潜水员卡片后，可查看：

- 该潜水员累计案例数
- 高频潜点
- 主要污染来源
- 汇总报告

### 4.10 LLM 连通性测试

管理员可点击：

- `测试 LLM 连通性`

系统会返回：

- 是否连接成功
- 当前模型
- 当前接口地址
- 失败原因

## 6. 常见问题

### 6.1 手机打不开页面

检查：

1. 手机和电脑是否在同一局域网
2. 是否使用了 `http://电脑IP:8000`
3. 是否错误使用了 `https://`

### 6.2 手机定位失败

说明：

- 这通常是浏览器安全策略导致
- 在 HTTP 页面下常见

处理：

1. 先点 `读取手机定位`
2. 失败后改用 `地图选点`

### 6.3 LLM 未调用成功

管理员端模型配置区域会显示：

- 最近错误
- 最近成功时间
- 当前接口地址

建议先点击：

- `测试 LLM 连通性`

### 6.4 提交慢

原因通常是：

- 当前正在跑图像增强与识别
- 正在调用远程模型

系统已加入加载提示，请等待处理完成。
