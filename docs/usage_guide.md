# SeeSea 快速上手

> 当前 GitHub 仓库暂不公开源码。本页用于快速了解系统操作链路和演示顺序。

## 1. 系统入口

系统包含两个用户端：

- 潜水员端：现场采集、上岸复核、称重分类、提交反馈。
- 管理员端：态势总览、活动管理、垃圾审核、任务分配、漂流分析、报告发布。

同一套响应式 Web 支持：

- 手机移动端。
- 桌面网页端。
- 主机端/大屏展示。

## 3. 推荐演示链路

### 步骤 1：管理员查看治理态势

进入管理员端后，查看：

- 治理地图。
- KPI 指标。
- 活动趋势。
- 潜水员贡献表。
- 志愿者心声洞察。

### 步骤 2：潜水员水下采集

切换到潜水员账号：

1. 进入水下模式。
2. 选择潜点。
3. 完成签到。
4. 记录环境信息。
5. 拍照或上传图片。
6. 地图选点。
7. 提交生成案例和垃圾身份证。

### 步骤 3：潜水员岸上复核

1. 进入岸上模式。
2. 在日历中选择日期。
3. 查看 AI 初判。
4. 对分类、数量、重量和清理状态进行复核。
5. 确认入库。
6. 提交志愿者心声。

### 步骤 4：管理员审核垃圾身份证

1. 进入垃圾管理。
2. 打开某个垃圾身份证。
3. 查看原图、AI 增强图、AI 识别图。
4. 查看 AI 证据链。
5. 审核通过或重新审核。
6. 对未清理垃圾生成任务。

### 步骤 5：管理员分配任务

1. 进入任务分配。
2. 选择已有未清理垃圾，或在地图上选择新的任务地点。
3. 指派潜水员。
4. 设置优先级、风险等级和截止日期。
5. 创建任务。

### 步骤 6：运行漂流溯源沙盘

1. 进入漂流沙盘。
2. 选择单个垃圾或区域范围。
3. 选择溯源 + 扩散模式。
4. 运行沙盘分析。
5. 查看来源轨迹、未来扩散、概率区和区域样本。

### 步骤 7：生成多智能体报告

1. 进入报告中心。
2. 选择报告类型。
3. 生成多智能体报告。
4. 查看视觉、证据链、溯源、安全、行动规划和公益影响 Agent 结论。
5. 确认发布。
6. 打印/导出为课件。

## 4. 核心展示重点

- 垃圾不只是识别框，而是结构化身份证。
- AI 结论都有证据来源和人工复核状态。
- 地图可以支撑治理决策，而不是只做展示。
- 志愿者反馈可以转为安全和流程优化任务。
- 报告中心把技术结果转化为项目复盘和公益传播材料。
