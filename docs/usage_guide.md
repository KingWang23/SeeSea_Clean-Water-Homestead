# SeeSea 使用指南

## 1. 适用对象

本指南适合以下使用场景：

- 你第一次启动和体验 SeeSea
- 你要快速跑通一次管理员端 + 潜水员端的闭环演示
- 你要给队友一份简明的操作手册

## 2. 启动系统

```bash
cd <project-root>
conda activate clean_underwater_demo
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

其中 `<project-root>` 为项目根目录。

访问地址：

- 本机：`http://127.0.0.1:8000/`
- 手机：`http://你的电脑IP:8000/`

注意：

- 手机访问时必须使用 `http://`
- 浏览器若自动跳转到 `https://`，请手动改回 `http://`

## 3. 默认账号

- 管理员：`admin / admin123`
- 潜水员 A：`diver_a / diver123`
- 潜水员 B：`diver_b / diver123`

## 4. 一次完整演示怎么跑

### 4.1 管理员端

1. 打开登录页
2. 用管理员账号登录
3. 查看首页指标、地图、趋势图和 AI 指挥简报
4. 保持管理员页打开，用于后续查看新增案例

### 4.2 潜水员端

1. 在另一浏览器或手机上登录潜水员账号
2. 填写：标题、潜点、日期、位置描述
3. 录入经纬度
4. 若手机定位失败，则点击 `地图选点`
5. 填写人工表单和志愿者反馈
6. 点击 `拍照上传` 或 `选择图片`
7. 提交案例

### 4.3 回到管理员端

1. 点击刷新后台
2. 查看新案例
3. 打开案例详情
4. 展示原图、增强图、识别图、核对结果和分析报告
5. 演示审核状态修改或导出报告

## 5. 角色能力速览

### 潜水员端

- 拍照/选图
- 经纬度填写
- 地图选点
- 人工表单录入
- 志愿者反馈录入
- 查看案例结果
- 删除本人案例

### 管理员端

- 看板查看
- 案例筛选
- 风险预警
- AI 指挥简报
- 工作流审核
- 新建潜水员
- 数据导出
- LLM 连通性测试

## 6. 常见问题

### 6.1 手机定位失败

原因通常是：

- 当前页面是 `HTTP`
- 浏览器拒绝 `navigator.geolocation`

处理方式：

- 直接使用 `地图选点`

### 6.2 管理员和潜水员在同一浏览器里切换会不会冲突

不会。

当前系统已使用角色独立 token 鉴权，管理员和潜水员登录态不会互相覆盖。

### 6.3 LLM 没有成功调用怎么办

管理员端可通过“模型配置状态”和“测试 LLM 连通性”排查：

- 当前接口地址
- 最近错误
- 最近成功时间
- 超时、重试和模型配置

### 6.4 浏览器提示 `/favicon.ico 404`

这是浏览器自动请求网站图标，没有实际功能影响，可忽略。
