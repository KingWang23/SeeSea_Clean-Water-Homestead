# SeeSea 使用教程

## 1. 系统概览

SeeSea 是一个面向海洋垃圾治理的双端系统：

- 潜水员端：上传现场图片、位置、人工表单、志愿者反馈
- 管理员端：查看案例、审核状态、生成报告、分析潜点态势
- AI 服务：图像增强、垃圾识别、表单核对、扩散分析、语义分析、指挥简报

## 2. 启动方式

在项目目录执行：

```bash
cd "海洋马拉松/Project"
conda activate clean_underwater_demo
uvicorn app.main:app --port 8000
```

说明：

- 长时间联调时，建议不要使用 `--reload`
- 默认地址为 `http://127.0.0.1:8000`

## 3. 登录流程

1. 打开首页 `/`
2. 输入管理员或潜水员账号
3. 系统会自动进入对应工作台

注意：

- 首页现在固定是登录页，不会再因为旧会话自动跳管理员
- 如果同一浏览器里频繁切换管理员/潜水员，建议先退出登录

## 4. 潜水员端使用流程

### 4.1 提交一个新案例

1. 登录潜水员账号
2. 填写实例标题、潜点名称、拍摄日期
3. 填写位置描述与经纬度
4. 如使用手机，可点击“读取手机定位”
5. 填写天气、能见度、人工表单备注
6. 填写志愿者语义反馈
7. 新增人工表单条目
8. 拍照或上传图片
9. 点击“提交本次实例”

### 4.2 系统自动完成的工作

提交后系统会自动执行：

1. 图像增强
2. AI 垃圾识别
3. 表单核对
4. 志愿者反馈分析
5. 扩散路径模拟
6. 报告生成

### 4.3 潜水员可查看的内容

- 原图 / 增强图 / AI 识别图
- 识别与人工核对结果
- 实例分析报告
- 管理员反馈
- AI 补充洞察

### 4.4 删除案例

- 进入“我的上传实例”
- 选择案例
- 点击“删除本案例”

系统会同步删除：

- 案例数据库记录
- 未被其他案例复用的图片文件

## 5. 管理员端使用流程

### 5.1 首页看板

管理员登录后可查看：

- 实例总数
- 活跃潜水员数
- AI 识别垃圾数量
- 平均清晰度增益
- 风险分布
- 月度上传趋势
- 地图分布

### 5.2 AI 指挥简报

管理员可根据当前筛选条件生成：

- 行动摘要
- 重点潜点
- 传播建议
- 数据质量关注点
- 下一步动作

页面会直接显示：

- 是否调用了远程 LLM
- 若未命中远程 LLM，是否已回退到本地规则

### 5.3 实例处理

选择左侧实例后可执行：

- 查看详情
- 查看原图/增强图/识别图
- 重新生成分析
- 删除案例
- 导出 Markdown
- 导出 JSON

### 5.4 审核工作流

管理员可以维护：

- 审核状态
- 清理状态
- 优先级
- 管理员备注

### 5.5 潜水员管理

管理员可以：

- 查看潜水员汇总
- 新增潜水员账号
- 查看潜水员历史上传与汇总报告

## 6. LLM 状态说明

如果页面显示：

```text
local-heuristic / rule-based-dashboard-v1
```

表示本次没有成功使用远程 LLM 结果，而是回退到了本地规则生成。

常见原因：

1. 没配置 `SILICONFLOW_API_KEY`
2. 远程接口超时
3. 返回内容不是有效 JSON
4. 接口地址不是官方推荐地址，建议使用 `https://api.siliconflow.com/v1`

当前系统会在管理员端模型配置面板中额外显示：

1. 最近成功调用时间
2. 最近一次错误信息
3. 超时秒数、重试次数和输出上限

如果管理员端和潜水员端在同一个浏览器中切换使用，本系统现在采用角色独立 token 鉴权，不需要依赖共享 session，所以不会再因为另一端重新登录而频繁出现 `403 Forbidden`。

如果页面显示：

```text
siliconflow / deepseek-ai/DeepSeek-R1-Distill-Qwen-7B
```

表示本次确实使用了远程文本模型。

## 7. 建议的演示顺序

1. 登录管理员端，展示总览看板
2. 进入潜水员端，提交一个案例
3. 等待 AI 处理完成
4. 回到管理员端查看新案例
5. 展示 AI 指挥简报
6. 展示识别图、核对结果、扩散路径和报告
7. 修改审核状态并展示潜水员汇总

## 8. 常见问题

### 8.1 页面显示“数据库正忙”

说明当前已有长任务在处理，请等待后重试。

### 8.2 页面长时间没有响应

优先检查：

- 是否使用了 `--reload`
- 是否正在调用远程 LLM
- 网络是否正常

### 8.3 为什么同一浏览器会切换角色后出问题

因为当前使用基于 session 的登录方式，同一个浏览器会话只保存一个角色。

建议：

- 管理员和潜水员分别使用不同浏览器
- 或一个普通窗口 + 一个无痕窗口
