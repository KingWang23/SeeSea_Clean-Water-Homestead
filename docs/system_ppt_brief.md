# SeeSea 系统介绍与 PPT 讲解提纲

## 1. 项目定位

SeeSea 是一套面向海洋垃圾治理行动的双端智能协同系统，目标是把潜水员现场上传的图片和表单，自动转化为可追踪、可核验、可分析的海洋垃圾证据数据。

核心价值：

1. 让“图像即数据”
2. 让“垃圾有身份证”
3. 让“案例可追踪、可核验、可汇总”
4. 让“管理决策有 AI 辅助”

## 2. 解决的问题

当前公益清理行动中常见问题：

1. 水下图像质量差，难以直接识别
2. 人工表单容易漏填、错填
3. 垃圾来源判断依赖经验，难标准化
4. 潜点数据分散，缺少统一协作平台
5. 志愿者现场经验难沉淀为结构化知识

SeeSea 针对这些问题，构建了“采集端 + 管理端 + AI 分析链路”的完整闭环。

## 3. 系统总体架构

系统由四层组成：

### 3.1 采集层

- 潜水员手机拍照或上传图片
- 填写潜点、位置、时间、人工表单
- 补充志愿者语义反馈

### 3.2 AI 处理层

- 图像增强
- 多模型垃圾识别
- OCR 与包装特征辅助判断
- 表单核对
- 志愿者反馈语义分析
- 扩散路径模拟
- LLM 报告与指挥简报生成

### 3.3 管理层

- 管理员查看案例
- 审核与清理状态维护
- 潜水员汇总管理
- 数据导出与报告留档

### 3.4 展示层

- 地图总览
- 趋势统计
- 风险预警
- 指挥简报
- 单案例分析报告

## 4. 系统功能模块

## 4.1 水下图像增强

目标：

- 修复偏色
- 提升清晰度
- 降低噪声

当前实现：

- Gray World 白平衡
- CLAHE 局部对比度增强
- 双边滤波
- 锐化与去噪

价值：

- 为后续识别提供更稳定输入
- 提高展示效果与科研可用性

## 4.2 垃圾“身份证”识别

输出字段包括：

- 垃圾名称
- 类别
- 材质
- 来源活动
- 尺寸等级
- 数量
- 状态
- 置信度
- 证据来源

当前识别链路：

1. 传统视觉规则
2. OCR 文本提取
3. 包装模板匹配
4. 可选本地 ONNX 检测器
5. SiliconFlow GLM 视觉模型辅助识别
6. 多源结果融合

创新点：

- 不依赖单一模型
- 支持证据链展示
- 识别结果可回溯

## 4.3 人工表单核对

系统自动对比：

- AI 识别总数
- 人工表单总数
- 漏填项
- 数量不一致项

价值：

- 减少手工统计误差
- 提高行动数据可信度

## 4.4 志愿者反馈语义分析

输入：

- 志愿者文本反馈

输出：

- 垃圾密集度
- 环境复杂度
- 安全风险等级
- 关键词提取
- 简易报告模板

价值：

- 把经验性描述转为可管理数据
- 丰富行动后评估维度

## 4.5 垃圾来源与扩散分析

系统基于：

- 垃圾类型
- 潜点位置
- 来源活动判断

生成：

- 扩散路径模拟
- 来源倾向分析
- 管理建议

说明：

- 当前为规则驱动模拟
- 后续可接入真实海流与潮汐模型

## 4.6 管理员 AI 指挥简报

管理员可基于当前筛选条件生成：

- 行动摘要
- 重点潜点
- 传播建议
- 数据质量关注点
- 下一步动作

意义：

- 把案例级数据提升为行动级决策支持
- 支持公益传播与资源调度

## 5. 系统流程

完整流程如下：

1. 潜水员在移动端登录
2. 上传现场图片、位置与表单信息
3. 系统执行图像增强
4. 系统执行垃圾识别与核对
5. 系统执行反馈分析与扩散模拟
6. 系统生成单案例报告
7. 管理员在后台查看案例与地图态势
8. 管理员生成 AI 指挥简报
9. 管理员更新审核状态与清理状态
10. 系统对潜水员数据生成汇总报告

## 6. 创新点总结

可以在 PPT 中重点强调以下几点：

### 6.1 图像即数据

把潜水员拍摄的原始图像自动转为结构化治理数据。

### 6.2 垃圾有“身份证”

每件垃圾不只是被看见，还能被识别、分类、核验与追踪。

### 6.3 AI 与人工双向校验

不是用 AI 替代人，而是用 AI 帮助人纠错、补全、提效。

### 6.4 双端协同

移动端采集，后台管理，形成完整闭环。

### 6.5 从案例级走向行动级

不仅输出单案例结果，还输出潜点态势、风险提醒和指挥简报。

## 7. 使用的技术栈

### 7.1 后端

- FastAPI
- SQLAlchemy
- SQLite

### 7.2 前端

- HTML / CSS / JavaScript
- Leaflet 地图

### 7.3 AI / 算法

- OpenCV 图像增强
- OCR 可选接入 Tesseract
- 本地 ONNX 检测器
- SiliconFlow：
  - `THUDM/GLM-4.1V-9B-Thinking`
  - `deepseek-ai/DeepSeek-R1-Distill-Qwen-7B`

### 7.4 工程机制

- Session 登录鉴权
- 文件上传与图像缓存
- Markdown / JSON / CSV 导出
- 前端请求超时与加载态提示
