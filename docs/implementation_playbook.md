# 实施手册（归档参考）

> 本文件保留早期黑客马拉松阶段的实施拆解思路，仅作为历史归档和扩展参考。当前以以下文档为准：

> - `docs/usage_guide.md`
> - `docs/detailed_usage_manual.md`

## 1. 早期目标拆解

1. 图像增强：提高水下图像可辨识度
2. 垃圾身份证：输出类别、材质、来源、数量、状态
3. 表单核对：自动发现漏填与数量偏差
4. 协作平台：网页展示 + 移动端提交 + 后台分析
5. 反馈语义：提取密集度、复杂度、安全风险
6. 溯源模拟：基于位置和来源类型推演扩散路径

## 2. 早期架构草图

- 后端：FastAPI + SQLAlchemy + SQLite
- AI 服务层：
  - `enhancer.py`
  - `identifier.py`
  - `reconcile.py`
  - `trace.py`
  - `feedback_nlp.py`
- 前端：
  - 管理员 Web 页面
  - 潜水员移动端页面
  - 地图与可视化组件

## 3. 早期可交付 Demo 标准

1. 能上传图片并看到增强前后效果
2. 能输出垃圾识别结果
3. 能与人工表单核对
4. 能在地图中看到潜点位置
5. 能生成一条轨迹模拟结果
6. 能生成一份分析报告

## 4. 与当前正式版本的关系

当前正式版本已经在上述基础上补全了：

- 双角色登录与权限隔离
- 移动端拍照上传与地图选点
- 管理员审核工作流
- 导出能力
- LLM 连通性测试
- 更完整的文档体系

## 5. 赛后扩展方向

1. 替换更强的图像增强模型
2. 替换更强的视觉检测与 OCR 组合
3. 接入真实海流数据与漂移模型
4. 升级数据库和存储系统
5. 扩展为多机构协同版本
