# SeeSea 功能与需求匹配说明

## 1. 项目方核心需求

项目方提出的核心目标可以概括为：

- 用 AI 分析潜水员提交的水下垃圾图片和视频。
- 识别垃圾种类、材质、大小、包装文字和图片信息。
- 判断垃圾可能来源于什么人类活动。
- 预测垃圾扩散路径，生成可视化溯源模拟报告。
- 建立网页展示、小程序/手机表单、后台管理、数据库和数据分析面板的协作平台。
- 支持图像增强、来源辨识、志愿者反馈语义分析和沉浸式可视化。

SeeSea 的系统设计围绕这些需求展开，并进一步加入垃圾身份证、AI 证据链、漂流溯源沙盘和公益行动闭环。

## 2. 需求与功能对应关系

| 需求 | 系统功能 | 满足方式 |
| --- | --- | --- |
| 水下图像增强 | 原图、AI 增强图、AI 识别图 | 对水下偏色、模糊、低对比图像进行增强展示，预留 FUnIE-GAN 接口 |
| 垃圾分类识别 | 垃圾身份证 | 记录 100 类分类、材质、数量、尺寸、风险等级和清理状态 |
| 包装文字识别 | OCR 证据链 | OCR 文字作为来源活动和类别判断证据 |
| 来源活动判断 | 来源分析与风险分析 | 结合类别、材质、OCR、现场备注和知识图谱推断来源 |
| 扩散路径预测 | 漂流溯源沙盘 | 支持单件垃圾和区域范围的溯源/扩散分析 |
| 数据管理平台 | 管理员端 CRM | 管理潜水员、垃圾、任务、审核、报告和人员 |
| 潜水员表单提交 | 潜水员端 | 手机网页完成签到、拍照、定位、分类、称重和反馈 |
| 志愿者反馈分析 | 心声语义分析 | 提取垃圾密集度、环境复杂度、安全隐患和流程问题 |
| 成果展示 | 治理地图与报告中心 | 用地图、趋势图、贡献表和报告课件展示公益成效 |

## 3. 潜水员端功能

### 3.1 日历工作台

- 按日期查看历史工作记录。
- 查看当天作业批次、案例、任务和审核状态。
- 汇总工作时间、上传案例数、清理重量和任务完成量。

对应需求：项目方希望记录潜水员活动、历史数据和项目成果。

### 3.2 水下模式

- 入水签到。
- 安全确认。
- 选择潜点和关联任务。
- 记录天气、能见度、水流、深度和坐标。
- 支持拍照或上传图片。
- 地图选点记录垃圾坐标。
- 每张图片生成一个案例并进入数据库。

对应需求：满足潜水作业流程中入水前标准程序、水下拍摄、位置记录和现场采集要求。

### 3.3 岸上模式

- 按日期选择待复核案例。
- 查看原图、AI 增强图、AI 识别图。
- 修改分类、数量、重量和清理状态。
- 录入称重方式。
- 确认入库或重新提交复核。
- 提交志愿者心声。

对应需求：满足出水后称重、垃圾分类、计数、数据记录和志愿者反馈分析要求。

## 4. 管理员端功能

### 4.1 治理态势总览

- 真实地图展示垃圾案例坐标。
- 展示案例数、垃圾件数、清理重量和待审核数。
- 活动趋势图展示近 14 天作业变化。
- 潜水员贡献表展示每位潜水员的案例、重量和任务贡献。
- 志愿者心声洞察用于识别安全和流程问题。

### 4.2 潜水员活动管理

- 查看潜水员列表。
- 点击潜水员进入工作日历。
- 查看每日上传案例、作业批次和任务状态。

### 4.3 垃圾管理

- 以垃圾身份证为核心管理对象。
- 区分已清理和未清理垃圾。
- 展示原图、增强图和 AI 识别图。
- 展示 AI 证据链。
- 管理员可审核通过、重新审核或生成清理任务。

### 4.4 任务分配

- 可基于未清理垃圾生成任务。
- 可在地图上选择新的任务地点。
- 任务点位会写入空间坐标，便于潜水员执行和管理员查看。

### 4.5 漂流溯源沙盘

- 支持选择单个垃圾进行分析。
- 支持选择一片区域进行分析。
- 区域选点时显示垃圾分布。
- 输出反向溯源轨迹、未来扩散轨迹、概率区、区域样本和行动建议。

### 4.6 报告中心

- 一键生成多智能体治理报告。
- 报告包含 KPI、证据快照、漂流引用、任务联动和志愿者反馈。
- 支持草稿、确认发布、归档和打印/导出课件。

## 5. 核心创新

### 5.1 垃圾身份证

每件垃圾不再只是图片上的识别框，而是一条结构化档案，包含分类、材质、重量、数量、坐标、风险、来源、状态、证据和任务关系。

### 5.2 AI 证据链

系统记录每个判断的来源，包括图像增强、YOLO、OCR、VLM、大模型、规则融合和人工复核，解决 AI 黑盒问题。

### 5.3 漂流溯源沙盘

地图从展示工具升级为治理决策工具，用于判断垃圾可能来源、未来扩散方向和优先治理区域。

### 5.4 公益行动闭环

系统覆盖发现、识别、审核、派发、清理、称重、报告和传播，帮助公益行动从活动记录升级为可复盘的数据资产。
