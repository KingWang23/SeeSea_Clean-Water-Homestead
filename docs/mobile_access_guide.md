# SeeSea 手机访问部署说明

本说明适用于：

- 你的台式电脑作为服务器
- 手机与电脑连接同一局域网
- 手机直接访问潜水员端或登录页

## 1. 前提条件

确保：

1. 台式电脑和手机连接同一个 Wi-Fi / 局域网
2. 电脑上的服务已经启动
3. 防火墙允许访问你设置的端口

## 2. 推荐启动方式

进入项目目录后执行：

```bash
cd "/home/stf/比赛/CCF 海洋马拉松/Project"
conda activate clean_underwater_demo
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

说明：

- `127.0.0.1` 只能本机访问
- `0.0.0.0` 表示局域网内其他设备也可访问

## 3. 查看电脑局域网 IP

在 Linux 终端执行：

```bash
hostname -I
```

会得到类似：

```bash
192.168.31.45
```

记住这个地址。

## 4. 手机访问方式

在手机浏览器输入：

```text
http://192.168.31.45:8000
```

其中：

- `192.168.31.45` 替换成你电脑实际 IP
- `8000` 替换成你实际启动端口

## 5. 推荐访问入口

手机上建议直接访问：

- 登录页：`http://你的IP:8000/`
- 潜水员端：登录后自动进入

不建议手机直接长期操作管理员端，因为管理员端信息密度更高，适合电脑大屏。

## 6. 防火墙问题

如果手机打不开页面，优先检查防火墙。

若你使用 `ufw`，可执行：

```bash
sudo ufw allow 8000/tcp
```

然后再次访问。

## 7. 手机端推荐使用方式

### 7.1 提交流程

1. 手机浏览器打开登录页
2. 登录潜水员账号
3. 填写潜点和表单信息
4. 点击“读取手机定位”
5. 点击拍照或选择相册
6. 提交案例

### 7.2 注意事项

- 图片建议小于 10MB
- 网络较慢时，提交后会看到“正在上传图片并调用 AI 分析，请稍候...”
- 如果连接远程 LLM，等待时间会比本地规则更长

## 8. 若页面无法打开，按这个顺序排查

1. 电脑服务是否已启动
2. 启动命令是否使用了 `--host 0.0.0.0`
3. 电脑与手机是否在同一 Wi-Fi
4. IP 地址是否填写正确
5. 端口是否被防火墙拦截

## 9. 演示建议

推荐你在答辩或展示时这样操作：

1. 台式电脑投屏展示管理员端
2. 手机展示潜水员端
3. 手机拍照提交一个案例
4. 电脑端刷新后展示新案例进入后台分析链路

这样可以直观体现“移动采集 + 后台协同”的双端闭环。
