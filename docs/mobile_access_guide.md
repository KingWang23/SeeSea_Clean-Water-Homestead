# SeeSea 手机访问说明

## 1. 使用场景

本说明适用于：

- 台式电脑作为服务器
- 手机与电脑连接同一局域网
- 手机直接访问登录页或潜水员端

## 2. 正确启动方式

```bash
cd <project-root>
conda activate clean_underwater_demo
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

其中 `<project-root>` 为项目根目录。

说明：

- `127.0.0.1` 只允许本机访问
- `0.0.0.0` 才允许局域网设备访问

## 3. 查看电脑局域网 IP

```bash
hostname -I
```

输出示例：

```text
192.168.31.45
```

手机应访问：

```text
http://192.168.31.45:8000/
```

## 4. 注意不要使用 HTTPS

当前 `uvicorn` 默认只提供 HTTP 服务。

如果手机访问成：

```text
https://192.168.31.45:8000/
```

会出现：

- 手机提示“此网站无法提供安全连接”
- 终端出现 `Invalid HTTP request received`

这是因为客户端拿 HTTPS 去访问了 HTTP 端口。

正确方式必须是：

```text
http://你的电脑IP:8000/
```

## 5. 手机端推荐操作

### 5.1 进入系统

1. 在手机浏览器打开登录页
2. 登录潜水员账号
3. 进入潜水员端

### 5.2 提交案例

1. 填写基本信息
2. 点击 `拍照上传` 或 `选择图片`
3. 点击 `读取手机定位` 或 `地图选点`
4. 填写人工表单和志愿者反馈
5. 提交案例

### 5.3 定位说明

在手机浏览器中，`HTTP` 页面往往无法直接使用系统定位。

SeeSea 已提供双模式：

- 浏览器定位
- 地图选点

若定位失败，直接使用地图选点即可。

## 6. 打不开页面时的排查顺序

1. 电脑服务是否已启动
2. 启动命令是否使用 `--host 0.0.0.0`
3. 手机和电脑是否在同一 Wi-Fi
4. 是否使用了正确的电脑局域网 IP
5. 是否错误使用了 `https://`
6. 防火墙是否拦截 8000 端口

## 7. 防火墙说明

如使用 `ufw`，可放行 8000 端口：

```bash
sudo ufw allow 8000/tcp
```

## 8. 推荐答辩演示方式

- 电脑展示管理员端
- 手机展示潜水员端
- 手机现场拍照提交一个案例
- 电脑端刷新后台并展示完整分析链路

这样最能体现双端协作和真实使用场景。
